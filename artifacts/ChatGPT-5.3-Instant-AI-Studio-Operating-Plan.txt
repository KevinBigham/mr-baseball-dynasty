3. The Golden Development Loop
This is the most important section.
Every feature should follow this exact cycle.
Step 1
Start a new chat for every major task.
Examples:
• implement draft system
• improve trade UI
• optimize simulation engine
• build save system
Why?
Long chats cause reasoning decay.
Your 200k token suspicion is actually very close to reality.
Reasoning quality starts dropping around 150k–200k tokens in most models.
So yes:
New task = new chat.
________________


Step 2
Architect writes TASK.md
Example structure:
Goal
Add scouting system to Mr Football Dynasty


Context
Files involved:
engine/playerEngine.ts
systems/scoutingSystem.ts


Constraints
Do not modify engine RNG logic
Follow architecture rules in AGENTS.md


Done When
Scouting reports generate
Tests pass
UI displays scouting grades
This file lives in the repo.
________________


Step 3
Give Codex the task
Codex reads:
• TASK.md
• AGENTS.md
• repo files
Codex writes code.
________________


Step 4
Codex produces HANDOFF.md
This includes:
• files changed
• commands run
• results
• risks
• next steps
________________


Step 5
Claude reviews
Claude checks:
• architecture boundaries
• bugs
• performance risks
• regressions
Claude may suggest improvements.
________________


Step 6
Commit to GitHub
Once Claude approves, push the changes.
________________


Step 7
Start the next task in a new chat.
This keeps reasoning fresh.
________________


4. Managing AI Context Windows
Your YouTube observation is correct.
Even when models support huge contexts, reasoning deteriorates in long threads.
Use this rule:
Short chats (<20k tokens)
Safe.
________________


Medium chats (20k–100k tokens)
Still good.
________________


Long chats (100k–200k tokens)
Start splitting tasks.
________________


Massive chats (>200k tokens)
Start new chat immediately.
________________


The trick
The repository itself becomes memory.
AI chats stay short.
Repo files stay long.
That’s the key.
________________
